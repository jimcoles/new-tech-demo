SVG Static

<svg>: Scalable Vector Graphics (SVG). Supports the SVG graphics model for relative graphics contexts, path (line, curve, polygon) drawing with path closure. Includes some high-level graphics objects. Good for creating clickable diagrams. 

            (absolute, relative)
            M, m - move
            L, l - line to
            H, h - horiz line
            V, v - vert line
            C, c - cubic bezier
            S, s - smooth cubic bezier
            Q, q - quadratic bezier
            T, t - smooth quadratic bezier
            A, a - elliptical arc
            Z, z - close path

Pros

  • Used by Google Drawing, which features highly editable box and line user objects.
  • SVG objects are known elements to the browser. Can be given id's and event handlers.

Cons:

  • We'd get no help with 3D effects.

Canvas Static

<canvas> : Low-level graphics

Canvas is a low-level bitmap graphics object for 2D and 3D. We'd like to use this if possible so that we can get as sophisticated as possible.

Notes:

  • Used by Google Maps for the map itself, but requires the Google JS library for event handling) i.e., the browser does not know about the map objects or events.
  • Can use the WebGL library.
  • 2D support. 3D not supported by Safari.
  • Bitmap drawing so the browser does not know about the 'objects' you're drawing.
